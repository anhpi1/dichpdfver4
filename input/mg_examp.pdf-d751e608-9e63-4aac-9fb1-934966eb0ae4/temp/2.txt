MAP GRID EXAMPLE [17]
The files and directory structure for the map grid example are shown at the right. Copy these file to your hard drive and make note of the directory as this must be identified as part of the setup procedure. [18]
The directories are named heights and backdrop. The backdrop directory contains a scanned topographic map and ariel photo in the TIF file format. The heights directory contains a terrain database with the same extents as the scanned map. Each directory contains an index file and a projection file in the MSI Planet format. The index file identifies the file name and the edges of the file and this will be loaded into the program. The projection file states the required projection. [19]

Map Grid example files and directories [21]
The file mg_examp.bkd is a map grid project file which contains all of the required definitions for both the backdrop and the terrain database. [22]
The file mg_examp.gr4 is a Pathloss network data file which can be viewed in both the network and map grid modules. [23]
The first part of the example uses the map grid project file. The second part describes the steps to create the map grid project file and data sources and production issues. [24]

ELEVATION VIEWS [76]
An elevation view can be added to the display. [77]
Select Site data - Elevation view - Create. The display extents is always the current screen. [78]
The button toggles the elevation view on and off. [79]
The elevation view overlays the backdrop using raster operations. Several options for the raster operation codes are available. Select Site Data - Elevation view - Options. The SRCCOPY code shown at the right completely overwrites the backdrop. The DSPDxax code is a transparent display which is useful with 8 bit grey scale backdrop images. [80]
ADDING AND MOVING SITES [81]
Select Site data - Add site or Move site to access these features. [82]
The initial site locations and subsequent adjustments can be made with the add and move site operations. Moving a site requires two steps. First click on the site to be moved. Then move the site to its new position. [83]
Moving a site does not automatically regenerate the path profiles for any links connected to this site [84]
CREATING A MAP GRID PROJECT [85]
This section describes how to create the map grid project file for this example. [86]
Select Site data - Backdrop and click the reset button to clear the current project. [87]
In this release the same projection must be used for both the backdrop and the elevation data. Even if there is no backdrop you must set the projection for both. Starting with the backdrop proceed as follows. [88]
The file \backdrop\projectn.txt contains the line “NAD27 Clarke 1886 zone 10". These are the required parameters for a UTM projection. [89]

GEOTIF FILES [120]
GeoTif files are TIF image files with extensions to geographically reference the file. The map grid module supports GeoTif files with UTM and US state plane coordinate projections. The directory map_grid\Geo_Tiff contains an example of a GeoTif file using US state plane coordinates. [121]
Select Site data - Backdrop and click the reset button to create a new map grid project. In the backdrop section click the index button. [122]
Select Files - Geo Tiff file and load the file map_grid\geo_tiff\sp27gtif.tif. The information dialog at the right summarizes the geographic status of the file. [123]
Click the create index button. All aspects of the backdrop definition are filled in. [124]
Close the index and click the OK button to display the file. [125]
BACKDROP DATA SOURCES [126]
The file vcr_map.tif was produced by scanning a topographic map in several sections and joining up the sections in a image editing program. A Hewlett Packard ScanJet 3200C was used. The edges must be geo- referenced. This means that the rectangular coordinates of the edges must be known. This is a relatively simple procedure if the map contains a UTM grid. The edges of the image can be cut to the grid lines and these values will be entered into the index. [127]
The file format can be either a tagged image file (TIF) or windows bitmap (BMP). [128]
The color should be set to 8 bit palette for both color and grey scale images. The base file size will be one byte per pixel. If 24 bit color is used the base file size will be three times the size. [129]
